## Problem 1 — Word Embeddings from IIT Jodhpur Data

**File:** `assignment2_p1.py`  
**Models:** CBOW and Skip-gram with Negative Sampling (pure NumPy, no Gensim)

### Run everything end to end

```bash
python assignment2_p1.py
```

### Run individual tasks

```bash
python assignment2_p1.py --task 1   # scrape + preprocess + corpus stats + word cloud
python assignment2_p1.py --task 2   # train Word2Vec models (CBOW + SGNS)
python assignment2_p1.py --task 3   # nearest neighbours + analogy experiments
python assignment2_p1.py --task 4   # PCA and t-SNE visualizations
```

> **Note:** Tasks must be run in order. Task 2 needs `corpus.txt` from Task 1. Tasks 3 and 4 need the `.pkl` model files from Task 2.

### What each task does

| Task | Input | Output |
|------|-------|--------|
| Task 1 | IIT Jodhpur website (scraped live) | `raw_data/`, `corpus.txt`, `corpus_stats.json`, `wordcloud.png`, `top20_words.png` |
| Task 2 | `corpus.txt` | `cbow_model.pkl`, `sgns_model.pkl`, `task2_loss_curves.png`, `task2_results_table.png` |
| Task 3 | `cbow_model.pkl`, `sgns_model.pkl` | Printed results + `task3_neighbours.png`, `task3_analogies.png` |
| Task 4 | `cbow_model.pkl`, `sgns_model.pkl` | `task4_pca.png`, `task4_tsne.png`, `task4_visualization.png` |

### Hyperparameters (Task 2)

Configured at the top of `assignment2_p1.py`:

```python
EMBED_DIMS   = [50, 100]    # embedding dimensions to try
WINDOW_SIZES = [2, 5]       # context window sizes to try
NEG_SAMPLES  = [5, 10]      # negative samples per positive
EPOCHS       = 5
LR           = 0.025        # initial learning rate (decays linearly)
MIN_FREQ     = 2            # ignore words appearing fewer than this many times
```

To train a 300-dimensional model as required by the embedding vector task:
```python
EMBED_DIMS = [300]
```

### Scraper notes

The IIT Jodhpur main site (`iitj.ac.in`) is JavaScript-rendered and cannot be scraped with plain `requests`. The scraper uses `old.iitj.ac.in` (static HTML mirror) and `academics.iitj.ac.in` instead. If some URLs return empty pages, remove them from `SEED_URLS` at the top of the file — there are enough seed URLs that the corpus will still be adequate.

## Problem 2 — Character-level Name Generation

**File:** `assignment2_p2.py`  
**Models:** VanillaRNN, BidirectionalLSTM, AttentionRNN (PyTorch)

### Run everything end to end

```bash
python assignment2_p2.py
```

On subsequent runs, `TrainingNames.txt` is loaded from disk.

### Run individual tasks

```bash
python assignment2_p2.py --task 0   # generate TrainingNames.txt (skips if file exists)
python assignment2_p2.py --task 1   # train all three models
python assignment2_p2.py --task 2   # evaluate: novelty, diversity, qualitative analysis
```

> **Note:** Task 0 must run before Task 1. Task 2 needs the `.pt` checkpoint files from Task 1.

### What each task does

| Task | Input | Output |
|------|-------|--------|
| Task 0 | Claude API (one-time) | `TrainingNames.txt` (1000 Indian names) |
| Task 1 | `TrainingNames.txt` | `rnn_model.pt`, `blstm_model.pt`, `attn_model.pt`, `p2_loss_curves.png` |
| Task 2 | All three `.pt` files | `generated_VanillaRNN.txt`, `generated_BLSTM.txt`, `generated_AttentionRNN.txt`, `p2_evaluation.png`, `p2_eval_results.json` |

### Hyperparameters (Task 1)

Configured at the top of `assignment2_p2.py`:

```python
HIDDEN_SIZE = 256     # hidden state size for all models
N_LAYERS    = 2       # number of stacked layers
DROPOUT     = 0.3     # dropout between layers
EPOCHS      = 60
LR          = 0.001   # Adam learning rate
BATCH_SIZE  = 64
```

Reduce `EPOCHS` to 30 or `HIDDEN_SIZE` to 128 if training is too slow on CPU.

### GPU support

The code automatically uses CUDA if available:
```python
DEVICE = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
```

No changes needed to use GPU — just run normally and it will pick it up.

### Generated name files

After Task 2, three text files are created with generated names, one per line:
- `generated_VanillaRNN.txt`
- `generated_BLSTM.txt`
- `generated_AttentionRNN.txt`

## Troubleshooting

**Scraper returns empty pages / "too short"**  
The URL structure of iitj.ac.in changes occasionally. Open the URL in a browser, check if it loads content, and update `SEED_URLS` in `assignment2_p1.py` if needed.

**NLTK data not found error**  
```bash
python -c "import nltk; nltk.download('punkt'); nltk.download('punkt_tab'); nltk.download('stopwords')"
```

**Word not found in vocabulary (print_embedding.py)**  
The word may have been filtered due to `MIN_FREQ=2`. Try a more frequent word, or lower `MIN_FREQ` and retrain.

**BLSTM training too slow on CPU**  
Lower `HIDDEN_SIZE` to 128 and `EPOCHS` to 30 in `assignment2_p2.py`. This reduces params but still produces reasonable output.

## Notes

- All Word2Vec code (Problem 1) is implemented from scratch using NumPy only — no Gensim or PyTorch used.
- Problem 2 uses PyTorch's `nn.RNN`, `nn.LSTM`, `nn.GRU` modules with custom attention layer.
- The name generation API call runs exactly once. Re-running the pipeline does not regenerate or overwrite `TrainingNames.txt`.
- Random seeds are fixed (`numpy.random.seed(42)`, `torch.manual_seed(42)`) for reproducibility.
